OpenClaw 到底是什么？

OpenClaw 是一个「本地优先」的 AI 智能体（Agent）框架，由奥地利开发者 Peter Steinberger 创建。它最核心的能力是：

工具调用（Tool Calling）：能真正去查你的 ERP、物流系统、电商后台 API，拿到真实数据再回答，不瞎编。
多 Agent 协同：可以同时跑「选品 Agent + 客服 Agent + 数据 Agent」，各司其职不打架。

全平台消息集成：可接入 WhatsApp、飞书、企业微信、Slack 等，卖家在哪里工作，助手就在哪里。
传统 AI 工具只能「管中窥豹」，只看到你输入的那几行字。而 OpenClaw 的 Agent 能读你整个业务系统的上下文——这才是质的飞跃。

🔍 AI 选品 Agent
让数据说话，不再靠感觉押注

痛点 vs 方案 选品场景
❌ 现在的苦
每天人工扒 AliExpress 榜单、手动记录 Google Trends，靠直觉判断哪款能卖，选错了仓库积货几十万。
✅ Agent 怎么做
指定关键词 + 品类，Agent 自动抓多平台销量趋势、竞品评论情绪、搜索热度曲线，输出带评分的选品报告。

📋 选品 Agent 核心 Prompt 模板（可直接用）

System Prompt 你是一位跨境电商选品分析师。每次用户输入品类或关键词，你需要： 1. 调用 web_search 工具检索近 30 天该品类在 Amazon/Shopee/TikTok Shop 的销量趋势 2. 调用 trends_api 获取 Google Trends 数据，识别上升期品类 3. 爬取 Top 20 竞品的评论，用情绪分析提取用户真实痛点 4. 输出结构化报告，格式如下： ## 选品评估报告 - 市场热度评分（1-10） - 竞争激烈度（低/中/高） - 用户核心痛点（TOP 3） - 差异化切入建议 - 风险提示 # 注意：禁止凭空猜测，所有结论必须基于调用工具的真实数据
💡 干货 Tip

选品 Agent 接入竞品 ASIN 爬虫后，可以自动分析差评中重复频率最高的词（如 "fragile", "wrong size"），这就是你的产品改良方向。结合 Helium10 API，效果翻倍。

📋 电商客服 Agent 四层架构

意图识别 Agent：判断是「物流查询 / 退换货 / 优惠咨询 / 投诉」，路由到对应的专项 Agent
物流查询 Agent：接入 ERP/物流 API，实时返回真实快递状态，不编造
售后处理 Agent：按退换货政策自动判断资格，符合条件直接走流程，超权限转人工
升级 Agent：检测到负面情绪（愤怒、威胁差评）时，立即转真人 + 打优先标签
核心配置思路 # 关键：给客服 Agent 设置「安全边界」 退款权限控制：AI 只能发起申请，实际退款由业务系统二次确认 敏感操作：修改收货地址、大额退款需短信验证码 数据隔离：每个店铺 Agent 只能访问本店 token，不交叉 审计日志：所有操作留痕，支持事后追溯 # 多语言配置（跨境必备） languages: ["zh-CN", "en-US", "ja-JP", "id-ID", "th-TH"] auto_detect: true fallback: "en-US"

📋 数据 Agent 核心监控指标配置

监控配置示例 # 电商数据 Agent 巡检规则 alerts: - name: "退货率异常" condition: return_rate > 8% # 超过行业均值 action: push_feishu + 生成差评归因报告 - name: "库存预警" condition: stock_days < 14 action: push_wechat_work + 触发补货建议工作流 - name: "广告 ROI 下跌" condition: acos_7d > acos_30d * 1.2 action: 暂停低效广告组 + 推送优化建议 - name: "爆款信号" condition: sales_3d_growth > 40% action: 触发快速补货 + 广告加价流程
💡 运营黄金法则

现有 AI 助手几乎都是「你问我答」模式——库存告急？你不问就不知道。

OpenClaw 的 Agent 可以配置成「主动巡检 + 主动预警」，这才是真正的数字运营助手，而不是一个等着被问的聊天机器人。

建议优先配置：退货率、库存天数、ACOS 三个核心指标的自动预警。
